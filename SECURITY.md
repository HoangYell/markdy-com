# Security Policy

## Supported Versions

Please use the latest version of Markdy packages to ensure you have the latest security patches. We generally only provide security updates for the current major release.

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We prioritize the security of our users and their applications. If you discover a security vulnerability within Markdy, we appreciate your help in disclosing it to us in a responsible manner.

Please **do not** open a public GitHub issue for security vulnerabilities. Instead, report it privately:

1. Send an email to **hoangyell@gmail.com**.
2. Include a clear description of the issue.
3. Provide steps to reproduce the vulnerability.
4. If applicable, specify the version(s) of Markdy affected.

### Triage Process

1. We will acknowledge receipt of your vulnerability report within 48 hours.
2. We will investigate the issue and determine its severity and impact.
3. We will collaborate with you to verify the vulnerability and develop a fix.
4. We will release a security update addressing the vulnerability.
5. Once the fix is published, we will publicly acknowledge your contribution (unless you request to remain anonymous).

Thank you for helping to keep the Markdy ecosystem secure!
